## ğŸ“ Database Normalization

### First Normal Form (1NF)
- All tables have primary keys
- No repeating groups
- Atomic values in each field

### Second Normal Form (2NF) 
- Meets 1NF requirements
- No partial dependencies
- All non-key attributes depend on entire primary key

### Third Normal Form (3NF)
- Meets 2NF requirements  
- No transitive dependencies
- All attributes depend only on primary key

### Example: User Skills Table
```sql
-- BEFORE Normalization (Violates 1NF)
CustomUser Table:
- user_id: 1
- skills: "Python, JavaScript, React, Django"

-- AFTER Normalization (Follows 3NF)
Skills Table:
- id: 1, user_id: 1, skill_name: "Python", skill_level: 4
- id: 2, user_id: 1, skill_name: "JavaScript", skill_level: 5  
- id: 3, user_id: 1, skill_name: "React", skill_level: 3
- id: 4, user_id: 1, skill_name: "Django", skill_level: 4
```
